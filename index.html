Sessió 1 - 2 hores

1. Què és JUnit i per què existeix?

Imagina que has programat una funció que calcula l'àrea d'un cercle. Com saps que funciona correctament? Probablement executaries el programa, posaries uns quants valors i miraries si el resultat és correcte.

Però què passa quan el teu projecte té 50 funcions? O 500? Fer proves manuals és lent, avorrit i propens a errors.

La solució: JUnit

JUnit és un framework que permet automatitzar les proves del teu codi Java. Escrius una vegada els tests, i pots executar-los tantes vegades com vulguis en segons.

JUnit 5 és la versió més moderna i està dissenyat per aprofitar les característiques de Java 8+. És l'eina més popular per proves unitàries en Java.

2. Què és un Test Unitari?

Definició

Un test unitari és una prova que verifica que una unitat de codi (normalment una funció o mètode) funciona correctament de forma aïllada.

Característiques d'un bon test unitari:

  • Específic: Prova una sola cosa
  • Ràpid: S'executa en mil·lisegons
  • Repetible: Sempre dona el mateix resultat
  • Aïllat: No depèn d'altres tests
  • Clar: Si falla, és obvi què ha anat malament

3. Caixa Negra vs Caixa Blanca

Caixa Negra (Black Box)

No mirem el codi intern

Testem la funció com un usuari: només ens importa que donada una entrada, obtenim la sortida esperada.

Exemple: "Si dono un 8, la nota ha de ser 'Notable'"

Caixa Blanca (White Box)

Mirem el codi intern

Testem que tots els camins del codi (if, else, bucles...) funcionen correctament. Ens assegurem que cobrim tots els casos.

Exemple: "Aquest if ha de ser cert almenys una vegada"

En aquesta sessió...

Farem testing de caixa negra. Ens donaran funcions ja fetes, i haurem d'escriure tests que verifiquin que funcionen correctament sense mirar com estan implementades per dins.

4. Anatomia d'un Test: Les 3 A

Tots els tests segueixen un patró molt clar anomenat AAA (Arrange, Act, Assert):

1

Arrange

Prepara les dades

2

Act

Executa la funció

3

Assert

Verifica el resultat

Exemple visual amb codi:

@Test public void testSuma() { // 1. ARRANGE: Preparem les dades int a = 5; int b = 3; // 2. ACT: Executem la funció int resultat = Calculadora.suma(a, b); // 3. ASSERT: Verifiquem que és correcte assertEquals(8, resultat); }

Consell important

Si un test és confús, pregunta't: "On està l'Arrange, l'Act i l'Assert?" Aquesta estructura ha d'estar sempre clara.

5. Els Asserts de JUnit

Els asserts són les eines que fem servir per comprovar que el resultat és l'esperat. JUnit 5 en té molts, però començarem amb els 3 més bàsics:

  • assertEquals(esperat, actual) - Comprova que dos valors són iguals
  • assertTrue(condició) - Comprova que una condició és certa
  • assertFalse(condició) - Comprova que una condició és falsa

Exemple d'ús:

@Test public void testNotes() { // assertEquals: valors iguals assertEquals(10, 5 + 5); // assertTrue: condició certa assertTrue(7 > 5); // assertFalse: condició falsa assertFalse("Hola".equals("Adéu")); }

6. AssertThrows: Testejant Errors

A vegades volem comprovar que la nostra funció llança un error quan les dades són incorrectes. Per això fem servir assertThrows.

Per què testejar errors?

És tan important verificar que el codi funciona bé amb dades correctes com que falla correctament amb dades incorrectes.

Exemple:

@Test public void testDivisioPerZero() { // Esperem que llanci una ArithmeticException assertThrows(ArithmeticException.class, () -> { int resultat = 10 / 0; }); }

Si la funció NO llança l'error esperat, el test fallarà.

DEMO EN VIU

Validador de Notes

Vegem un exemple complet pas a pas

Pas 1: La funció a testejar (ja feta):

public class ValidadorNotes { public static String qualificacio(int nota) { if (nota < 0 || nota > 10) { throw new IllegalArgumentException("Nota invàlida"); } if (nota < 5) return "Suspès"; if (nota < 7) return "Aprovat"; if (nota < 9) return "Notable"; return "Excel·lent"; } }

Pas 2: Els tests (seguint AAA):

import static org.junit.jupiter.api.Assertions.*; import org.junit.jupiter.api.Test; public class ValidadorNotesTest { @Test public void testNotaSuspes() { // Arrange int nota = 3; // Act String resultat = ValidadorNotes.qualificacio(nota); // Assert assertEquals("Suspès", resultat); } @Test public void testNotaAprovat() { assertEquals("Aprovat", ValidadorNotes.qualificacio(6)); } @Test public void testNotaNotable() { assertEquals("Notable", ValidadorNotes.qualificacio(8)); } @Test public void testNotaExcellent() { assertEquals("Excel·lent", ValidadorNotes.qualificacio(10)); } @Test public void testNotaInvalida() { // Comprovem que llança error amb nota -1 assertThrows(IllegalArgumentException.class, () -> { ValidadorNotes.qualificacio(-1); }); // Comprovem que llança error amb nota 11 assertThrows(IllegalArgumentException.class, () -> { ValidadorNotes.qualificacio(11); }); } }

Què hem cobert?

Amb aquests 5 tests, hem cobert:

  • Totes les qualificacions possibles (Suspès, Aprovat, Notable, Excel·lent)
  • Els casos de notes invàlides (menors que 0 o majors que 10)
